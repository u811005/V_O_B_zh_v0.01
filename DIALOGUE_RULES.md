# DIALOGUE_RULES.md

この文書は、`Village of Bacchus` の会話・セリフを追加または修正するときの運用ルールです。より細かい実装メモは `docs/dialogue-refactor-handoff.md` にあります。

## 基本方針

- セリフは原則として、対象キャラクター本人の発話にする。
- 状況説明だけで終わらせず、本人がどう感じているかを入れる。
- 口調は肉体ではなく精神側を優先する。
- 肉体状態、仕事、年齢、性別が通常とは異なる組み合わせでも破綻しない文にする。
- 世界観は神話、開拓村、村落生活、祈り、魔素、奇跡、祭、畑、森、酒場など既存語彙に寄せる。
- 現代的なサービス名、端末名、ネット文化、メタ発言は、強い意図がない限り避ける。

## 置き場所

通常会話は、できるだけ `js/data/dialogue/` 配下へ追加します。

- `statusLines.js`: 健康、疲労、消耗、襲撃中など。
- `conditionLines.js`: 負傷、病気、飢餓、凍え、精神状態など。
- `seasonLines.js`: 春、夏、秋、冬の季節会話。
- `jobLines.js`: 仕事に応じた会話。
- `reproductionLines.js`: 妊娠、臨月、産褥、成人化など。
- `exchangeLines.js`: 肉体交換への反応。交換の奇跡と落雷による肉体交換で共有する。
- `visitorLines.js`: 訪問者タイプ別会話。
- `randomEventLines.js`: ランダムイベント用の口調別セリフ。
- `toneProfiles.js`: 口調キー、alias、fallback、口調解決。

恋愛、結婚、奇跡結果など、一部の専用モーダル用セリフはまだ処理ファイル内に残っています。修正時は表示経路を確認し、既存構造に合わせてください。
肉体交換への反応は `exchangeLines.js` を更新し、交換の奇跡側とランダムイベント側で内容が分岐しないようにしてください。

## 口調キー

保存データにある基本値は `speechType` です。ただし、会話表示では精神特性から仮想口調キーを算出します。

主な口調キー:

- 子供: `赤子`, `男児`, `女児`
- 成人男性: `普通Ｍ`, `丁寧Ｍ`, `強気Ｍ`, `乱暴`, `お調子者`, `陰気`, `クールＭ`
- 成人女性: `普通Ｆ`, `丁寧Ｆ`, `お嬢様`, `快活`, `内気`, `強気Ｆ`, `蓮っ葉`, `おっとり`, `ぶりっこ`, `クールＦ`, `ギャル風`, `中性的`
- その他: `老人`

`spiritSex` と口調キーの対応:

| 区分 | 口調キー |
| --- | --- |
| 精神男性側 | `普通Ｍ`, `丁寧Ｍ`, `強気Ｍ`, `乱暴`, `お調子者`, `陰気`, `クールＭ`, `老人` |
| 精神女性側 | `普通Ｆ`, `丁寧Ｆ`, `お嬢様`, `快活`, `内気`, `強気Ｆ`, `蓮っ葉`, `おっとり`, `ぶりっこ`, `クールＦ`, `ギャル風`, `中性的` |
| 仮想子供口調 | `赤子`, `男児`, `女児` |

注意:

- `赤子` は精神特性 `無垢` から算出する。
- `赤子` は生後 6 か月程度の自然さを基準にし、意味のある単語や説明的な発話は避ける。喃語、泣き声、唸り声、寝息、必要最小限のしぐさ説明で状況を出す。
- `男児` / `女児` は精神特性 `萌芽` と精神性別から算出する。
- `赤子` / `男児` / `女児` は保存データに書き込まない。
- `中性的` は女性口調側のキーとして扱う。
- `老人` は明確に精神男性側の口調として扱う。

## 口調解決

通常会話の実効口調は `resolveDialogueTone(character)` で決まります。

1. `mindTraits` に `無垢` があれば `赤子`。
2. `mindTraits` に `萌芽` があれば、`spiritSex` に応じて `男児` または `女児`。
3. それ以外は `speechType` を正規化して使う。
4. `speechType` がなければ、`spiritSex` に応じて `普通Ｍ` または `普通Ｆ`。

ランダムイベントの成人向けセリフでは、保存値としての `speechType` を優先する経路があります。通常会話とランダムイベントでは参照経路が少し違うため、修正時は呼び出し元を確認してください。

## fallback の使い方

fallback は便利ですが、男女差や当事者性が混線しやすい場所です。

- 妊娠、産褥、仕事、身体変化など、話者によって自然な反応が変わる場面では実口調キーを直接書く。
- 状況キーを定義した通常会話では、原則として実効口調キーごとの明示行を用意し、fallback は例外除けとして扱う。
- `polite`, `bold`, `rough`, `cool`, `bright`, `male`, `female` などの共有 fallback には、誰が読んでも破綻しない文だけを置く。
- 既存データには古い fallback 中心の書き方も残っている。追加・修正時は、可能な範囲で実口調キー方式へ寄せる。

## セリフ作成のチェックリスト

- 本人の発話になっているか。
- 仕事や状態を、外から説明するだけの文になっていないか。
- 精神年齢・精神性別と口調が合っているか。
- `赤子` が意味語を話していないか。必要なら括弧でしぐさだけを補っているか。
- 肉体交換や特殊な状態でも不自然すぎないか。
- 現代的すぎる単語が混ざっていないか。
- 同じ内容のセリフが別ファイルに重複していないか。
- `default` や fallback に、特定の性別・年齢・肉体状態にしか合わない文を置いていないか。
- 配列が空のとき、会話エンジン側で安全に fallback できるか。

## 実装時の注意

- 通常会話は `getConversationLine({ character, village })` を入口にする。
- 専用シーンの 1 行取得は `getDialogueLine({ character, scene, key, context })` を使う。
- セリフ選択の優先度は `CONVERSATION_PRIORITY` に従う。
- `襲撃者` や `訪問者` は口調ではなく状態・役割として扱う。
- `妊娠`, `臨月`, `産褥`, `病気`, `負傷`, `季節`, `仕事`, `イベント種別` は口調キーにしない。
- 新しい scene を追加する場合は、データファイル、`getDialogueLines`、呼び出し元の表示経路を一緒に確認する。
- セリフ内に実装メモやデバッグ用の注釈を混ぜない。

## レビュー観点

会話変更のレビューでは、文法や誤字だけでなく次を確認します。

- そのセリフを読む人物が本当にその状態の当事者か。
- 口調 fallback で別性別・別年齢の人物に読まれても成立するか。
- 世界観から浮いた語彙がないか。
- 保存済みの `speechType` と仮想口調キーを混同していないか。
- ランダムイベント、通常会話、専用モーダルのどの経路で使われる文か明確か。
