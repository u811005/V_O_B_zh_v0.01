# CODE_REVIEW.md

この文書は、このリポジトリで変更レビューを行うときの観点をまとめたものです。レビューでは、見た目や好みよりも、ゲーム進行の破綻、保存互換性、状態不整合、セリフの混線を優先して確認します。

## レビューの姿勢

- 重大な不具合、データ破損、進行不能、保存互換性の破壊を最優先で探す。
- 指摘は、影響、再現条件、該当ファイルを具体的に書く。
- 好みのリファクタは、実害がない限り必須指摘にしない。
- 既存の設計やファイル分割に沿っているかを見る。
- 変更範囲外の未コミット差分は、依頼がなければレビュー対象に含めない。

## 重点チェック

### 月次進行

- `onNextTurn()` の処理順序が崩れていないか。
- 襲撃中、ゲームオーバー中、迎撃結果処理中のガードが効いているか。
- 月末処理、年月更新、年始の加齢、月初イベントの順番が意図通りか。
- 村人が 0 人になったときに以後の処理が不自然に続かないか。

### 状態と保存

- `Village` / `Villager` の新規フィールドに初期値があるか。
- 古い保存データに存在しないフィールドを読んでも壊れないか。
- JSON 保存と localStorage 保存で同じ状態を復元できるか。
- 人間関係、口調、仕事候補など、読込後に正規化が必要な値を処理しているか。
- 肉体情報と精神情報を誤って上書きしていないか。

### UI

- `updateUI(theVillage)` の呼び忘れがないか。
- PC 表示と `スマホ仮` 表示の両方で、表、ボタン、モーダルが崩れないか。
- テーブルの列番号や表示順に依存している処理が壊れていないか。
- ボタンの表示名、`data-mobile-label`、イベントハンドラが対応しているか。
- モーダルを閉じた後に状態やオーバーレイが残らないか。

### 会話・セリフ

- 口調解決が精神側の情報を優先しているか。
- `赤子` / `男児` / `女児` を保存値として書き込んでいないか。
- fallback に、特定の性別・年齢・肉体状態でしか成立しないセリフを置いていないか。
- セリフが本人の発話になっているか。
- 通常会話、ランダムイベント、専用モーダルの経路を混同していないか。
- 詳細は `DIALOGUE_RULES.md` を参照する。

### 襲撃

- 襲撃中の行動候補が `raidRules.js` と `jobTables.js` で一致しているか。
- 迎撃者なし、罠作成のみ、襲撃者なし、村人なしの境界値で壊れないか。
- 襲撃モーダルの進行状態が二重実行されないか。
- 襲撃完了後、村特性やフラグが適切に戻るか。

### ランダムイベント・奇跡

- 対象者がいない場合に安全に終了するか。
- 対象者の条件判定が肉体側・精神側のどちらを見るべきか明確か。
- 奇跡のコスト、効果、ログ、UI 更新が揃っているか。
- 交換や成長のような広範囲変更で、顔画像、口調、人間関係、仕事候補が更新されるか。

### 辞書データ

- 新しい仕事、行動、資源、建築、奇跡、状態異常、特性、イベント、襲撃者、ルールなどが、必要に応じて `js/data/dictionaryData.js` に追加されているか。
- 既存仕様を変更した場合、辞書の説明、数値、条件、効果が古いまま残っていないか。
- 用語検索でプレイヤーが確認する情報と、実際の処理内容が矛盾していないか。

### ドキュメント

- 新機能、仕様変更、ファイル構成変更、会話ルール変更、レビュー手順変更などが、影響する Markdown ドキュメントに必要に応じて反映されているか。
- 小さなバグ修正や内部実装だけの変更に、不要なドキュメント更新が混ざっていないか。

## 検証手順

変更内容に応じて、次を実施します。

- JavaScript 変更: 変更した `.js` ファイルを構文チェックする。
- ターン処理変更: 数か月進め、ログ、資源、村人状態、季節、イベント発生を確認する。
- 保存変更: JSON 保存/読込と localStorage 保存/読込を確認する。
- UI 変更: PC 表示と `スマホ仮` 表示を確認する。
- 会話変更: 通常会話、訪問者、状態異常、仕事、ランダムイベントなど該当シーンを確認する。
- 襲撃変更: 襲撃開始、迎撃割り振り、迎撃進行、結果処理を確認する。
- 新要素・仕様変更: `js/data/dictionaryData.js` と用語検索結果を確認する。
- 仕様・構造・運用ルール変更: 影響する Markdown ドキュメントを確認する。

## 指摘の重大度

- P0: データ破損、進行不能、起動不能、主要導線が完全に使えない。
- P1: よく起こる条件でゲーム進行や保存が壊れる。
- P2: 特定条件で不整合や明確な誤動作が起きる。
- P3: 小さな表示崩れ、誤字、将来の保守性低下。

## レビュー結果の書き方

指摘は重要度順に書きます。各指摘には、該当ファイル、何が起きるか、なぜ問題か、どう直すとよいかを含めます。問題が見つからない場合も、確認した範囲と未確認のリスクを短く残します。
